## EXP-001 — 하단 탭 내비게이션 도입

**배포일**: 2026-03-29  
**담당**: 단독  
**상태**: 🟡 진행 중 (배포 완료, 데이터 수집 중)

### 배경

기존 앱은 단일 화면 구조로, 설정이나 보상 같은 기능이 상단 버튼 하나에 숨겨져 있었다.  
신규 기능(보상, 더보기)을 지속적으로 추가할 계획인데, 현재 구조로는 기능 발견율(Feature Discovery Rate)이 사실상 0에 가깝다.

### 가설

> **하단 탭 내비게이션을 도입하면, 앱 내 보상/설정 기능의 발견율이 높아지고  
> 결과적으로 세션당 체류 시간과 재방문율이 개선될 것이다.**

### 변경 내용

| 항목 | Control (이전) | Treatment (이후) |
|------|--------------|----------------|
| 네비게이션 | 상단 우측 "설정" 버튼 | 하단 탭바 3탭 (홈 / 보상 / 더보기) |
| 접근 가능 기능 | 알람 목록 + 설정 | 알람 목록 + 보상 + 더보기(설정 포함) |
| 보상 페이지 진입 | 불가 | 탭 1클릭 |
| 설정 접근 | 알람 목록 → 설정 버튼 | 더보기 탭 직접 접근 |
| 시각적 현재 위치 | 없음 | 활성 탭 파란색 하이라이트 |

### 측정 지표

**Primary Metric**
- `/rewards` 페이지 방문율 (4월 MAU 대비 보상 페이지 방문 비율)

**Secondary Metrics**
- 탭 전환 발생 세션 비율 (홈 외 탭 접근 세션 %)
- 세션당 평균 페이지뷰 수 (이전 대비 증가 여부)
- 더보기 → 알림 설정 변경률

**Guardrail Metrics (악화 시 롤백 기준)**
- 알람 추가 완료율 — 탭 도입으로 혼란이 생길 경우 하락 가능
- 앱 이탈율 (첫 페이지 bounce)

### 성공 기준

| 지표 | 목표 |
|------|------|
| 보상 페이지 방문 세션 비율 | 배포 첫 주 **≥ 30%** |
| 더보기 탭 방문 세션 비율 | 배포 첫 주 **≥ 20%** |
| 알람 추가 완료율 | Control 대비 **유지 또는 개선** |

### 측정 방법

- 앱인토스 대시보드 페이지별 UV/PV 추적 (4월 데이터)
- 기준 비교: 3월 데이터 (설정 버튼 기간)
- 평가 시점: **배포 후 14일** (2026-04-12)

## EXP-002 — 10/30일 체크인 보상 시스템 도입

**배포일**: 2026-03-29  
**담당**: 단독  
**상태**: 🟡 진행 중 (배포 완료, 데이터 수집 중)  
**의존성**: EXP-001 (보상 페이지 탭 접근 가능해야 함)

### 배경

3개월 MAU 데이터에서 확인된 핵심 문제: **재방문 동기 없음**.  
현재 앱은 알람을 등록한 뒤 매칭이 일어나지 않으면 재방문할 이유가 없다.  
매칭 대기 기간 동안 유저를 붙잡을 **중간 리텐션 루프**가 필요하다.

### 가설

> **일일 체크인 기반 보상 시스템을 도입하면, 앱 재방문의 명시적인 동기가 생겨  
> D7 이상 리텐션율이 개선되고 4월 MAU 하락폭이 줄어들 것이다.**

### 변경 내용

| 항목 | Control (이전) | Treatment (이후) |
|------|--------------|----------------|
| 재방문 인센티브 | 없음 | 10일 / 30일 체크인 보상 |
| 10일 보상 | 없음 | 알람 슬롯 +1 (반복 가능) |
| 30일 보상 | 없음 | 광고 영구 제거 (1회) |
| "받기" 버튼 | 없음 | 항상 노출, 조건 충족 시 활성화 |
| 진행도 표시 | 없음 | `(N/10)`, `(N/30)` 배지 |

### 체크인 로직

```
방문 날짜 배열 → localStorage 저장 (고유 날짜 기준)
10일 체크인: 누적 방문일 ÷ 10 = 완료 사이클 수 (반복 가능)
30일 체크인: 누적 방문일 ≥ 30 → 1회 수령 가능
```

> 현재 구현: localStorage 기반 (클라이언트 단 추적)  
> 추후: 서버 API 연동 시 정합성 보장 필요 (`api.claimCheckin()`)

### 측정 지표

**Primary Metric**
- **D7 리텐션율** — 앱 첫 방문 이후 7일 이내 재방문 비율 (앱인토스 대시보드)

**Secondary Metrics**
- 체크인 완료율 (10일 보상 수령 유저 비율)
- 평균 재방문 간격 (일)
- 30일 보상 도달율
- 보상 수령 이후 알람 추가율 (슬롯 +1 → 실제 알람 추가로 이어지는지)

**Guardrail Metrics**
- 인앱 결제(슬롯 구매) 전환율 — 무료 슬롯 지급으로 인한 감소 여부 모니터링

### 성공 기준

| 지표 | 목표 |
|------|------|
| D7 리텐션율 | 3월 대비 **+10%p 이상** 개선 |
| 10일 체크인 완료 유저 비율 | 4월 활성 유저 기준 **≥ 15%** |
| 4월 MAU 하락폭 | 3월(162) 대비 **50% 이내** (≥ 81) |

### 측정 방법

- 앱인토스 대시보드 리텐션 코호트 분석
- 기준 코호트: 2026-03 신규 유저 D7 리텐션
- 비교 코호트: 2026-04 신규 유저 D7 리텐션
- 평가 시점: **2026-05-01** (4월 MAU 집계 후)

1. **Primary Metric 미달 시**: 2주 데이터 확인 후 롤백 또는 개선 후 재배포
2. **Guardrail 지표 악화 시**: 즉시 검토, 임계치 초과 시 롤백
3. **결과 해석 시 주의**: 현재 실험은 A/B가 아닌 Before/After 비교이므로 외부 요인(계절성, 바이럴 이벤트 등) 영향을 함께 고려
4. **최소 측정 기간**: EXP-001은 14일, EXP-002는 30일 이상 (체크인 특성상 단기 측정 무의미)
